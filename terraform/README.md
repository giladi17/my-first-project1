## ☁️ ניהול תשתית הענן (Terraform)

חלק זה בפרויקט אחראי על הקמה וניהול של תשתית הענן ב-AWS בצורה אוטומטית כקוד (IaC).

### 🛠 דרישות מוקדמות
לפני הרצת הקוד, יש לוודא שהסביבה המקומית שלך מוכנה:
1. **[AWS CLI](https://aws.amazon.com/cli/):** יש להוריד ולהתקין את כלי שורת הפקודה של AWS.
2. **[Terraform](https://www.terraform.io/downloads.html):** יש להוריד ולהתקין את טרפורם.
3. **קינפוג AWS:** הרץ בטרמינל את הפקודה `aws configure` והזן את ה-Access Key וה-Secret Key של החשבון שלך כדי לאפשר לטרפורם לגשת לענן.

### 🔄 תהליך ההקמה (Workflow)

כדי לאפשר עבודה מאובטחת ומסונכרנת, אנו שומרים את מצב התשתית (State) מרחוק ב-S3. לכן, תהליך ההקמה הראשוני מתבצע בשני שלבים:

1. **הקמת S3 Bucket:** תחילה מריצים את טרפורם *ללא* הגדרת ה-Backend המרוחק. פעולה זו מקימה את קופסת האחסון (Bucket) ב-AWS.
2. **הגדרת ה-Backend:** לאחר שה-Bucket מוכן, אנו מוסיפים לקוד את בלוק ה-`backend "s3"` ומאתחלים שוב. מעתה והלאה, טרפורם ישמור שם את קובץ ה-State.

### 💻 פקודות מרכזיות לשימוש
במהלך העבודה בתיקייה זו, אנו משתמשים בשלוש פקודות ליבה:

* **`terraform init` (אתחול):** הפקודה הראשונה שחובה להריץ. היא מכינה את סביבת העבודה המקומית, מורידה את התוספים הרלוונטיים (כמו ה-AWS Provider) ומחברת את ה-Backend ל-S3.
* **`terraform apply` (יישום):** הפקודה שקוראת את הקוד שלך, בודקת מה חסר בענן, ומקימה/מעדכנת את המשאבים בפועל (לאחר הקלדת `yes` לאישור).
* **`terraform destroy` (מחיקה):** פקודת ניקוי שמיועדת למחוק בבטחה את כל המשאבים שהוקמו על ידי טרפורם כדי למנוע עלויות מיותרות.
