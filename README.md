# פרויקט DevOps - פריסה ואוטומציה מלאה ב-AWS EKS

## 📌 סקירה כללית (Overview)
פרויקט זה מכיל פתרון מלא למשימת DevOps, המדגים הקמת תשתית בענן באופן אוטומטי, אריזת אפליקציית Flask בקונטיינר, פריסתה לאשכול EKS, ויצירת צינור CI/CD [1]. 
הפרויקט נבנה תוך הקפדה על שיטות עבודה מומלצות (Best practices), בדגש על ביצועים, ניהול עלויות ואבטחת מידע [3].

## 📂 מבנה התיקיות (Repository Structure)
* **`terraform/`**: קוד התשתית (IaC) להקמת סביבת ה-AWS [1].
* **`app/`**: קוד המקור של אפליקציית ה-Flask וקובץ ה-Dockerfile [2].
* **`helm/flask-app/`**: קבצי ה-Helm Chart המשמשים לפריסת האפליקציה על אשכול ה-EKS [3].
* **`.github/workflows/`**: תהליכי האוטומציה של GitHub Actions עבור האפליקציה ועבור התשתית (בונוס) [3].

---

## 🛠 1. הקמת התשתית ב-AWS (Terraform)
התשתית מנוהלת דרך Terraform ומקימה את המשאבים הבאים:
* באקט S3 לאחסון מצב המערכת (Terraform state) [1].
* רשת VPC הכוללת תתי-רשתות ציבוריות ופרטיות [1].
* אשכול EKS עם שתי קבוצות שרתים (Node Groups), אחת בכל תת-רשת [2]. (השתמשנו בשרתי `t3.micro` לחיסכון בעלויות).
* מאגר תמונות ECR לאחסון הקונטיינרים [2].
* תפקידי IAM והרשאות נדרשות [2].

**כיצד לפרוס את התשתית ידנית:**
1. היכנס לתיקיית התשתית: `cd terraform`
2. אתחל את סביבת הטרפורם: `terraform init`
3. צפה בתוכנית הפריסה: `terraform plan`
4. הפעל והקם את התשתית: `terraform apply -auto-approve`

*(הערה: פרויקט זה כולל גם את **בונוס 1** - אוטומציה של תהליך פריסת התשתית באמצעות GitHub Actions [3]).*
